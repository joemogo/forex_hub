# v8.0 — Strategy Center, Training Academy, Scanner fix, nav audit

A summarized record of what this release was asked to deliver and what actually shipped,
distilled from the release's own `APP_VERSION_LOG` entry in `index.html`. This is not a verbatim
transcript of the original request — see [RELEASE_NOTES.md](../RELEASE_NOTES.md) for the
condensed public summary of the shipped result.

## Scope

An autonomous UI/architecture release covering four named goals, none of which touched JVM or
ALEX trading methodology:

1. **Fix the Scanner page's bottom-panel layout clipping** — the Confluence/R:R Calculator/Alert
   Log row below the chart could be squeezed to nothing or clipped with no scrollbar.
2. **Rebuild the Rules page into a real Strategy Center** — replace the plain bullet-list page
   with a professional reference view built from the existing frozen strategy data (`RULES`,
   `WEIGHTS`, `ALERT_THRESHOLD`), including real, computed performance stats rather than
   fabricated ones.
3. **Audit every navigation item** — every top-nav leaf should route to a real page, or to an
   honest "not built yet" state, never to an unrelated existing page as a silent stand-in.
4. **Build a Training Academy foundation** — a structured learning-content section with named
   tracks and modules, at least some of which have real, substantial written content rather than
   placeholder text everywhere.

## What was delivered

- **Scanner layout**: root-caused as a fixed-height container plus `overflow:hidden` squeezing
  the lower panel; fixed by switching to `min-height` and letting the existing `.panel`
  scroll behavior reveal the rest. A second, pre-existing bug (the chart's own `flex:1` being
  dead CSS due to a non-flex parent) was found and fixed in the same pass.
- **Strategy Center**: `renderRules()` rewritten in place (call sites unchanged) into eleven
  sections — hero, thesis, setup-architecture flow, decision flowchart, entry model, risk
  framework, disqualifiers, checklist, trade management, a real `computeMogoStrategyPerformance()`
  performance section (honestly showing "insufficient clean sample" below 50 real trades rather
  than fabricating a figure), and a strategy-only version history. An ALEX tab was added showing
  an honest "Coming Soon" panel rather than an ALEX-specific writeup.
- **Navigation audit**: 7 nav leaves with no dedicated page (Charts, Analytics, Reports, Trade
  Inspector, Market Outlook, Preferences, Developer) were changed from silently opening an
  unrelated existing page to opening a shared, honest `comingSoonOpen()` panel naming what's
  planned and where the closest working functionality lives today.
- **Training Academy**: a new real nav leaf with an Academy Home, 5 tracks totaling 49 named
  modules, a full lesson-view layout with knowledge-check quizzes, and a 17-term glossary. 3
  Forex Foundations modules shipped with complete written content and a real quiz; the remaining
  46 honestly display "content coming in a future release." All Academy state lives under its
  own isolated `mogo_academy_progress` key.

## Constraints honored

- Zero JVM/ALEX signal, qualification, entry, stop, target, direction, sizing, or exit function
  touched.
- `RULES`/`RULES_ALEXG`/`WEIGHTS`/`ALERT_THRESHOLD`/`CHECKLIST` used only as read-only inputs.
- Academy state verified (via dedicated fixtures) to never read or write any trading-state
  variable or `localStorage` key.

## Verification

15 new fixtures, plus the complete pre-existing 278-fixture suite, unchanged. Manually verified
live in-browser at multiple viewport sizes for the Scanner fix, and through a full Academy
lesson → quiz → completion → reload-persistence flow. See
[TESTING.md](../TESTING.md) for the current testing approach (this release predates the durable
regression-baseline tool introduced in v9.0).
